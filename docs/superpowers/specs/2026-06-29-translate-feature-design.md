# 翻译功能设计文档

## 1. 功能概述

在主页面新增翻译功能，支持文本翻译和剪贴板翻译，使用 MyMemory API 提供免费翻译服务，翻译历史保存到 SQLite 数据库。

## 2. 核心功能

### 2.1 文本翻译
- 用户手动输入文本进行翻译
- 支持 18 种语言互译
- 翻译结果可一键复制

### 2.2 剪贴板翻译（两种触发方式）

#### 方式一：通知触发
- 复制文字后，通知右边显示翻译按钮
- 点击按钮自动打开翻译页面并填充文本
- 用户选择目标语言后点击翻译

#### 方式二：剪贴板历史记录触发
- 剪贴板历史记录每条记录右边显示翻译按钮
- 点击按钮自动打开翻译页面并填充文本

## 3. 页面设计

```

│  ⬅️ 翻译                           🔄 📋 │  ← 标题 + 操作按钮

├─────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐│
│  │                                     ││  ← 输入框（可从剪贴板自动填充）
│  │                                     ││
│  └─────────────────────────────────────┘│
│                                         │
│            [ 🔄 翻译 ]                   │  ← 翻译按钮（蓝粉渐变）
│                                         │
│  ┌─────────────────────────────────────┐│
│  │                                     ││  ← 结果框
```

### 3.2 样式设计
- 使用现有的蓝粉渐变配色（#3d8bff / #ff6ab0）
- 输入框和结果框使用圆角卡片样式
- 按钮使用渐变背景和悬浮效果
- 保持与剪贴板管理页面一致的风格

### 4.1 源语言列表（18种）

- en：英语
- ja：日语
- ko：韩语
- fr：法语
- es：西班牙语
- it：意大利语
- de：德语
- tr：土耳其语
- ru：俄语
- pt：葡萄牙语
- vi：越南语
- id：印尼语
- th：泰语
- ms：马来西亚语

### 4.2 目标语言映射
每种源语言有不同的目标语言支持列表（详见 API 文档）。

## 5. 翻译 API 配置

### 5.1 默认 API
- 使用 MyMemory API（免费）
- 基础 URL：`https://api.mymemory.translated.net/get`
- 无需 API Key

### 5.2 自定义 API（设置页面）
- 支持用户自定义 API 地址
- 支持配置 API Key（可选）
- 在设置页面 → 翻译设置中配置

## 6. 翻译历史

### 6.1 存储方式
- 使用 SQLite 数据库存储
- 数据库文件：`userData/translate.db`

### 6.2 数据表结构

```sql
CREATE TABLE translate_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source_lang TEXT NOT NULL,      -- 源语言代码
  target_lang TEXT NOT NULL,      -- 目标语言代码
  source_text TEXT NOT NULL,      -- 源文本
  result_text TEXT NOT NULL,      -- 翻译结果
  created_at INTEGER NOT NULL     -- 创建时间戳
);
```

### 6.3 历史功能
- 自动保存每次翻译记录
- 支持查看历史列表（分页）
- 支持删除单条记录
- 支持清空所有历史
- 支持点击历史记录重新翻译

## 7. 技术实现

### 7.1 新增文件

#### 前端（src/renderer/src/views/）
- `Translate.vue` - 翻译页面组件

#### 主进程服务（src/main/service/）
- `translateService.ts` - 翻译 API 调用和历史管理

### 7.2 修改文件

#### 主页面（src/renderer/src/views/MainPage.vue）
- 添加翻译菜单项到侧边栏
- 引入 Translate 组件

#### 通知窗口（src/renderer/src/views/NoticeNew.vue）
- 在通知右边添加翻译按钮
- 点击按钮发送 IPC 事件打开翻译页面

#### 剪贴板管理（src/renderer/src/views/ClipboardManager.vue）
- 每条记录右边添加翻译按钮
- 点击按钮发送 IPC 事件打开翻译页面

#### 设置页面（src/renderer/src/views/Settings.vue）
- 添加翻译 API 配置区域
- 支持配置 API 地址和 API Key

#### 主进程入口（src/main/index.ts）
- 初始化翻译服务
- 注册翻译相关 IPC 事件

### 7.3 IPC 通信

#### 翻译服务
- `to-service-TranslateService:translate` - 调用翻译 API
- `to-service-TranslateService:getHistory` - 获取翻译历史
- `to-service-TranslateService:delete` - 删除翻译记录
- `to-service-TranslateService:clearAll` - 清空翻译历史

#### 翻译页面
- `to-main-Translate:open` - 打开翻译页面并填充文本
- `to-renderer-Translate:fillText` - 主进程发送文本到翻译页面

## 8. 交互流程

### 8.1 从通知触发翻译
1. 用户复制文字
2. 通知窗口显示，右边有翻译按钮
3. 用户点击翻译按钮
4. 主进程收到 IPC 事件
5. 主进程打开主页面（如果未打开）
6. 主进程切换到翻译页面
7. 主进程发送文本到翻译页面
8. 翻译页面自动填充文本
9. 用户选择目标语言
10. 用户点击翻译按钮
11. 调用翻译 API
12. 显示翻译结果
13. 自动保存到翻译历史

### 8.2 从剪贴板历史触发翻译
1. 用户打开主页面
2. 切换到剪切板页面
3. 看到历史记录列表
4. 每条记录右边有翻译按钮
5. 用户点击翻译按钮
6. 主进程收到 IPC 事件
7. 主进程切换到翻译页面
8. 主进程发送文本到翻译页面
9. 翻译页面自动填充文本
10. 后续流程同上
